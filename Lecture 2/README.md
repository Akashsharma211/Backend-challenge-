5. REST Api: Representation State Transfer Api => It is an architectural style for designing networked applications. It uses standard HTTP methods (GET, POST, PUT, DELETE) to perform operations on resources identified by URLs.
GET => server se data fetch krwana ho tb use krenge
POST => server pr data send krwana ho tab use krte hain
PUT => server pr data update karwana ho tab use krte hain
DELETE => server pr data delete karwana ho tab use krte hain
PATCH => server par data already hai aur us data ko update krna ho tab use krte hain.
